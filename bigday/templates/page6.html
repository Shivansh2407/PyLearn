What are Errors in a code? And why do they occur?
Errors arise when the interpreter is not able to parse your code properly and is not able to decipher it according to the stated rules. Python programme terminates as soon as it encounters an error which can be either a run-time error or a compile time error. Let us take a closer look at what are the different types of errors and how to avoid them.

The two types of errors are:-

1. Compile-time error :- These errors occur before the code has been compiled and is running. The compiler hence stops the execution of the program and the programmer needs to rectify them. Eg: Syntax error, Logical error etc.
2. Run-time errors :- These errors occur once the code has been compiled and is running. These errors are hard to detect and can make your program to behave unexpectedly or even crash. Eg: Zero divide error, Endless Loop etc.


Types of Compile-Time errors you can encounter while writing a python code will be:-

1. Syntax Error:-
This Error occurs when the programmer violates the grammar or the semantics of the language. The syntax of a language is the most basic part of a language and for using all the in-built functions we must know the syntax of the language.

Example:
>>> print "hello"
SyntaxError: Missing parentheses in call to 'print'.Did you mean print("hello")?
In Python 3.x, print is a built-in function and requires parentheses. The statement above violates this usage and hence syntax error is displayed.


Types of Run-Time Errors you can encounter while writing a python code will be:-

1. Index Error arises when trying to access an item at an invalid index. Mostly, occurs when we use lists when we try to access any element outside the length of the list.

Example:
>>> L1=[1,2,3]
>>> L1[3]
Traceback (most recent call last):
File "<pyshell#18>", line 1, in <module>
L1[3]
IndexError: list index out of range

2. Module Not Found Error arises when a module could not be found.

Example:
>>> import bk4
Traceback (most recent call last):
File "<pyshell#10>", line 1, in <module>
import bs4
ModuleNotFoundError: No module named 'bk4’

3. Key Error arises when a key is not found in a dictionary.

Example:
>>> dict={'1':"a", '2':"b", '3':"c"}
>>> dict['4']
Traceback (most recent call last):
File "<pyshell#15>", line 1, in <module>
dict['4']
KeyError: '4'

4. Import Error is thrown when a specified function cannot be found in the given module.

Example:
>>> from math import cube
Traceback (most recent call last):
File "<pyshell#16>", line 1, in <module>
from math import cube
ImportError: cannot import name 'cube'

5. Stop Iteration is thrown when the next() function goes beyond the iterator items.

Example:
>>> it=iter([1,2,3])
>>> next(it)
1
>>> next(it)
2
>>> next(it)
3
>>> next(it)
Traceback (most recent call last):
File "<pyshell#23>", line 1, in <module>
next(it)
StopIteration

6. Type Error is thrown when an operation or function is applied to an object of an inappropriate type.

Example:
list=['a']
>>> no=13
>>> v=list[0]/no
Traceback (most recent call last):
  File "<pyshell#5>", line 1, in <module>
    v=list[0]/no
TypeError: unsupported operand type(s) for /: 'str' and 'int'

7. Value Error is thrown when a function's formal and actual parameter’s types don’t match.

Example:
>>> int('a')
Traceback (most recent call last):
File "<pyshell#14>", line 1, in <module>
int('a')
ValueError: invalid literal for int() with base 10: 'a'

8. Name Error arises when the programmers callas for an undeclared object.

Example:
>>> fun
Traceback (most recent call last):
File "<pyshell#9>", line 1, in <module>
fun
NameError: name 'fun' is not defined

9. Zero Division Error arises when a number is divided by zero.

Example:
>>> x=1/0
Traceback (most recent call last):
File "<pyshell#8>", line 1, in <module>
x=1/0
ZeroDivisionError: division by zero
